# 蜂箱（Beehive）玩法与设计文档

## 一、概述

蜂箱是本模组的核心工作方块，模拟蜜蜂从繁殖到产出的完整生命周期。玩家需要向蜂箱中放入蜂后、雄蜂和巢脾等特定物品，蜂箱会根据环境参数自动完成幼虫孵化、基因遗传、产物积累与生成等一系列流程。

蜂箱的工作由两个协作层完成：

- **顶层协调层**——负责槽位管理、工作模式控制、参数收集和合法性校验。
- **生命周期处理器**——负责具体的繁殖循环、基因运算和产物生成。

两者分工明确：顶层协调层决定"能不能开始"以及"何时开始"，生命周期处理器决定"如何运作"以及"产出什么"。

---

## 二、槽位系统

蜂箱内部包含四类槽位，每类槽位有严格的物品准入限制。

### 2.1 蜂后槽位

- 数量由具体的蜂箱方块类型决定（通常为 1 到 2 个）。
- 只能放入蜂后（Queen Bee）物品。
- 工作周期开始时，消耗一只蜂后作为繁殖起点，提取其基因组。
- 工作周期结束时：
  - 若蜂后槽中有幼虫，该幼虫会羽化为新的蜂后。
  - 若蜂后槽为空，则在该槽位生成一瓶蜂王浆（Royal Jelly）。
  - 若蜂后槽中没有幼虫且蜂箱中有幼虫，则报告"王台为空"错误并等待重试。

### 2.2 雄蜂槽位

- 只能放入雄蜂（Drone）物品。
- 工作周期开始时，消耗所有雄蜂以提取它们的基因组，作为父本参与后代基因重组。

### 2.3 巢脾槽位

- 只能放入巢脾（Comb）物品。
- 巢脾槽位是幼虫填充和产物输出的主要场所：
  - 繁殖过程中，新生成的幼虫会优先放入空的巢脾槽。
  - 幼虫在巢脾中随时间积累产量。
  - 工作周期结束时，巢脾中的幼虫会根据其积累的产量值转化为对应的最终产物（如蜜脾、丝等）。

### 2.4 额外槽位

- 预留给子类扩展使用。基础蜂箱中不接受任何物品。
- 不同的蜂箱变种可以通过额外槽位实现独特功能。

所有槽位底层共用同一个物品栈资源池，因此不同区域的槽位数量之和即为蜂箱的总物品容量。

## 三、工作模式

蜂箱支持三种工作模式，玩家可通过 GUI 随时切换：

### 3.1 手动模式（默认）

- 需要玩家手动触发启动信号（例如放入蜂后会触发"待工作"标记）。
- 每次工作周期结束后自动停止，需要再次手动触发才能开始下一轮。
- 适合精细操作和观察蜜蜂繁殖过程。

### 3.2 自动模式

- 只要环境条件满足、物品充足，蜂箱会自动循环工作。
- 工作周期结束后立即开始新一轮的合法性校验，通过后自动进入下一个周期。
- 适合大规模、自动化生产线。

### 3.3 红石模式

- 仅当蜂箱接收到红石信号时才会工作。
- 信号消失后，即使工作周期未完成也会暂停。
- 信号恢复后继续推进当前周期。
- 适合与红石电路联动实现定时或条件控制。

## 四、完整工作流程

一个完整的蜂箱工作周期分为以下六个阶段：

### 4.1 触发条件

触发工作周期的条件因工作模式而异：

- **手动模式**：玩家向空的蜂后槽位放入物品，系统自动标记"待工作"。
- **自动模式**：每个 tick 都标记为"待工作"。
- **红石模式**：收到红石信号的每个 tick 标记为"待工作"。

### 4.2 合法性校验

每次尝试启动前，系统会逐一检查以下条件。任意一条不满足都会拒绝启动，并在 GUI 上显示对应的错误信息：

1. **手动模式未触发**——手动模式下必须有启动信号。否则报"手动停止"。
2. **蜂后数量检查**——必须有且仅有一只蜂后。零只报"缺少蜂后"；多只报"蜂后过多"。
3. **蜂后槽位内容检查**——蜂后槽中不允许出现非蜂后物品。否则报"槽位格式错误"。
4. **雄蜂数量检查**——雄蜂槽和巢脾槽中的雄蜂总数至少为 1。不足报"缺少雄蜂"。
5. **雄蜂/巢脾槽位内容检查**——这两类槽中不允许出现非雄蜂物品。否则报"槽位格式错误"。

全部通过后，错误码设为"正常"，进入冷却阶段。

### 4.3 冷却倒计时

校验通过后，蜂箱不会立即开始工作，而是进入一个冷却倒计时（默认 100 tick）。冷却期间 GUI 会显示进度条。冷却完成后再执行繁殖启动流程。

冷却机制的作用：
- 给玩家一个确认和反悔的窗口期。
- 防止因物品快速进出导致的频繁启动。

### 4.4 启动繁殖周期

冷却结束后，系统在一个事务保护下执行以下操作：

1. **消耗蜂后**——从蜂后槽中取出一只蜂后，提取其基因组（二倍体，即两组基因）。
2. **消耗雄蜂**——依次从雄蜂槽和巢脾槽中取出所有雄蜂，提取每只雄蜂的基因组（单倍体，即一组基因），形成雄蜂基因池。
3. **收集外部参数**——从蜂后槽中物品携带的"插件"组件读取参数修饰（某些特殊物品可提供额外的速度、产量等加成）。如果物品标为消耗品，会尝试消耗该物品（如带有耐久的工具会扣除耐久）。
4. **构建环境参数集**——综合世界位置、生物群系、维度类型、方块状态等环境信息，加上从物品收集到的参数修饰，生成完整的参数集。
5. **环境验证**——检查蜂后的遗传信息是否适应当前环境（温度、湿度、生境偏好等）。不适宜则报"环境不适宜"。
6. **事务提交**——以上步骤全部成功后提交事务，将控制权转移给生命周期处理器。

### 4.5 繁殖与生产循环

生命周期处理器接管后，进入核心工作循环。该循环从蜂后的寿命基因和环境寿命参数计算出**总进度值**，然后逐 tick 减少进度，并在特定时刻触发不同行为。

#### 4.5.1 初始化

- 基于世界坐标和世界随机数生成一个可序列化的随机源（确保存档重载后结果一致）。
- 根据蜂后的繁殖力基因，随机计算出本轮将产生的幼虫数量和雄蜂数量。
- 设定工作总量 = 蜂后基础寿命 tick 数 × 环境寿命参数。

#### 4.5.2 进度推进

每 tick 根据**速度参数**（含随机浮动）减少当前进度值。当进度值跨过一个固定的"间隔"边界时，触发以下行为之一：

#### 4.5.3 第一阶段：幼虫填充

- 优先将幼虫放入空的巢脾槽位。
- 每个幼虫的基因组通过**基因重组**生成：从蜂后两组基因和随机选中一只雄蜂的基因组进行双倍体重组，再经过突变系统处理。
- 如果幼虫全部放完后还有未放完的名额，进入雄蜂填充阶段。

#### 4.5.4 第二阶段：雄蜂填充

- 将雄蜂放入空的雄蜂槽位。
- 雄蜂基因组通过**单倍体重组**生成：仅从蜂后基因组中随机选择一组，不进行突变。

#### 4.5.5 第三阶段：产品积累

当幼虫和雄蜂都已填充完毕后，进入产品积累阶段：

- 系统检测周围是否存在花朵。如果没有任何可识别的花，标记"无花"状态。
- 如果存在花，搜索周围的生境（Biotope）信息。
- 每个巢脾中的幼虫每间隔一段时间积累一次产量。产量计算公式为：
  - 基础值 = 幼虫的产量基因值 / （寿命系数 × 间隔长度） × 环境产量参数
  - 如果幼虫的生境偏好与当前环境中的生境匹配，产量按**匹配模式**增长。
  - 如果不匹配，产量按**通配模式**增长（效率较低），同时标记"无匹配生境"警告。
- 幼虫的过期时间会定期刷新，防止其在工作周期内腐败。

#### 4.5.6 蜂后选举（特殊机制）

在产品积累阶段，系统会周期性地检查蜂后槽位是否为空。如果有空位，执行一次"蜂后选举"：

- 遍历所有巢脾槽中的幼虫，计算每个幼虫的基因组与当前蜂后基因组的**相似度**。
- 选出相似度最高的幼虫，将其从巢脾槽位移入空的蜂后槽位。
- 这模拟了真实蜂群中工蜂选出接班蜂王的过程——与前任蜂后遗传特征最接近的幼虫优先继承王位。

#### 4.5.7 异常监测

前述阶段运行期间，系统每 tick 监测环境状态：

- 如果环境变得不适宜，停止进度推进，标记"环境不适宜"。
- 如果周围没有花，标记"无花"（产品积累暂停）。
- 如果部分幼虫的生境偏好不匹配，标记"无匹配生境"（仅影响产量，不停止工作）。

### 4.6 工作完成

当进度值归零时进入完成阶段：

1. **处理蜂后槽位**：
   - 若有幼虫，将其转化为蜂后（清除幼虫专用的产物和过期组件，保留基因组）。
   - 若没有幼虫但有空槽位，生成一瓶蜂王浆。
   - 若既没有幼虫又没有空槽位（所有槽位被堵塞），报告"王台为空"错误，进入短暂冷却后重试。

2. **处理巢脾产物**：
   - 所有巢脾槽位中的幼虫根据其积累的产量值，通过产物计算规则得出最终产物物品（蜜脾、丝或其他材料），替换掉幼虫。

3. **重置状态**：
   - 清空所有运行时状态（基因组缓存、计数、随机源等），为下一轮工作做好准备。

## 五、参数系统

蜂箱的工作受多种环境参数影响。这些参数通过一个可扩展的参数类型注册表统一管理。

### 5.1 内置参数

| 参数   | 默认值 | 说明                                       |
| ------ | ------ | ------------------------------------------ |
| 速度   | 1.0    | 影响每 tick 进度推进的速率                 |
| 突变率 | 1.0    | 影响后代基因发生突变的概率                 |
| 寿命   | 1.0    | 影响工作周期的总时长（蜂后实际寿命的乘数） |
| 产量   | 1.0    | 影响巢脾幼虫每次积累的产量值               |
| 温度   | 0.0    | 叠加到当前环境温度，用于环境适宜性判断      |
| 湿度   | 0.0    | 叠加到当前环境湿度，用于环境适宜性判断      |

所有内置参数的值类型均为浮点数，多个来源的同一参数通过**加法**合并。

### 5.2 参数来源

参数从多个来源汇聚，最终形成一个统一的参数集：

- **环境来源**：世界位置、生物群系、维度类型等自动生成基础参数。
- **物品插件**：某些特殊物品（如信息素）可以携带"插件"组件，放入蜂箱时提供额外的参数加成。这类物品分为"消耗型"（使用后消失或损耗耐久）和"非消耗型"（可重复使用）。
- **子类扩展**：不同的蜂箱变种可以通过自定义逻辑添加额外的参数来源。

### 5.3 参数格式化

参数值在 GUI 中以格式化形式显示：

- 速度、突变率、寿命、产量等比率型参数显示为**百分比格式**（如 `+20%`）。
- 温度、湿度等绝对值型参数显示为**数字格式**（如 `+2.0`）。

## 六、基因系统简述

虽然基因系统的完整设计在独立的基因包中，但蜂箱工作流程频繁使用基因机制，在此简述其角色。

### 6.1 基因结构

- 每只蜜蜂都有一套**基因组**，包含多个基因：产量基因、温度偏好基因、湿度偏好基因、生境偏好基因、繁殖力基因等。
- **蜂后**持有**双倍体基因组**——即两组完整的基因（一组来自母本，一组来自父本）。
- **雄蜂**持有**单倍体基因组**——只有一组基因。

### 6.2 遗传与重组

后代幼虫的基因通过以下方式确定：

- **幼虫（双倍体）**：从蜂后的两组基因中随机取一组，再从随机选中雄蜂的基因组中取一组，组合成新的双倍体。之后应用突变系统，有一定概率改变基因值。
- **雄蜂（单倍体）**：仅从蜂后基因组中随机选择一组，不经过突变。

### 6.3 环境适配

蜜蜂能否在当前环境正常工作，取决于其基因组中的以下基因是否与环境匹配：

- **温度偏好**：是否在环境温度的允许范围内。
- **湿度偏好**：是否在环境湿度的允许范围内。
- **生境偏好**：当前环境中的生境集合是否包含此蜜蜂偏好的生境。

只有三项全部匹配，环境才算"适宜"。其中生境不匹配仅作为警告处理（影响产量但不停止工作），温度和湿度不匹配则会直接暂停工作。

## 七、产物系统

### 7.1 产物积累

巢脾中的幼虫在每次产品积累触发时，根据以下因素增加值：

- **幼虫自身的产量基因**——基因值越高，每次增量越大。
- **环境产量参数**——来自参数集的产量倍率。
- **生境匹配度**——如果幼虫偏好的生境与当前环境匹配，使用匹配模式增长；否则使用通配模式增长（效率约减半）。

### 7.2 产物转化

工作周期结束时，巢脾中的幼虫被转化为最终产物。产物的具体类型由幼虫的基因型和当前世界环境共同决定。

### 7.3 蜂王浆

当蜂后槽位在工作结束时为空且没有可羽化的幼虫时，系统会在空槽位生成一瓶蜂王浆作为保底产出。

## 八、状态与错误码

蜂箱的当前状态通过 GUI 和容器数据接口向玩家展示。以下是所有可能的错误码及其含义：

| 错误码             | 含义                           | 玩家应对方法                                     |
| ------------------ | ------------------------------ | ------------------------------------------------ |
| OK                 | 一切正常                       | 无需操作                                         |
| MANUAL_HALT        | 手动模式，尚未触发工作         | 放入蜂后触发，或切换为自动模式                   |
| MISSING_QUEEN      | 缺少蜂后                       | 放入一只蜂后                                     |
| EXTRA_QUEEN        | 蜂后过多                       | 取走多余的蜂后，只保留一只                       |
| MISSING_DRONE      | 缺少雄蜂                       | 放入至少一只雄蜂                                 |
| MALFORMED_SLOT     | 槽位中有不该放的物品           | 检查各槽位，取出非法物品                          |
| NO_FLOWER          | 附近没有花                     | 在蜂箱周围种植花卉                                |
| NO_BIOTOPE         | 部分蜜蜂生境不符（警告）       | 可在当前环境工作，但产量降低                     |
| EMPTY_QUEEN        | 王台为空，无法产生新蜂后       | 清空巢脾或为蜂后槽腾出空间                       |
| INVALID_ENVIRONMENT | 环境不适宜                     | 调整环境（温度/湿度/维度）或更换适应该环境的蜜蜂 |

## 九、数据持久化

蜂箱的状态通过 NBT（命名二进制标签）持久化到世界存档中：

- **服务端完整存档**——保存所有槽位物品、生命周期处理器的全部状态（随机源种子、基因组缓存、计数、进度等）、插件参数、冷却进度、工作模式和错误码。
- **客户端同步**——仅同步冷却进度、工作模式和错误码，客户端不需要完整库存信息。
- **数据记录快照**——提供可序列化的数据记录，用于在不同方块实体之间传递完整状态，也用于网络数据包同步。

生命周期处理器内部使用**可序列化随机源**，确保所有随机事件（基因重组、突变、数量浮动等）在存档重载后结果一致。这意味着如果两次加载时环境和输入相同，蜂箱的行为是完全确定的。

## 十、设计理念总结

1. **模块化分层**——槽位管理、参数系统、生命周期处理各自独立，通过清晰的接口协作，降低了耦合度。

2. **可扩展性**——参数类型注册表支持在运行时注册自定义参数；额外槽位和自定义插件方法预留给子类扩展，使不同的蜂箱变种可以实现独特机制。

3. **事务安全**——所有关键的状态变更（消耗物品、生成后代）在事务中执行，失败时自动回滚，防止出现物品丢失等数据损坏问题。

4. **确定性随机**——所有随机行为使用可序列化随机源，以世界种子和方块位置为种子基础，确保存档一致性和结果可复现性。

5. **贴近真实的模拟**——蜂后选举、基因重组、突变、生境适配、寿命限制、产物积累等机制模拟了真实蜜蜂的生物学习性，为玩家提供沉浸式的养蜂体验。

6. **友好的玩家反馈**——通过详细的错误码系统，玩家可以快速定位蜂箱无法工作或效率低下的原因，降低了上手门槛。
